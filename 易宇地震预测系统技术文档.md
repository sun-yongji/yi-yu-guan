第一部分：系统概述

1.1 项目定位

易宇地震预测系统是易宇本源体系在地震预测领域的应用层实现。系统以全球主震后最大余震震级预测为目标任务，基于C₆六重对称群的时空编码框架，将传统干支纪年的时间相位调制、经度余弦的空间共振、以及区域构造敏感度三维度乘积，通过Båth定律的易学化扩展公式映射到余震震级估计值。

需要特别说明的是，本系统与太极矩阵工具包（taichi-shared / taichi-matrix，代号M0-M6）在代码层面无直接调用关系。虽然预测公式中使用的洛书修正系数0.034与taichi-shared中LUOSHU_CORRECTION的定义值相同，但地震预测脚本中全部硬编码，未通过import引用taichi-shared的任何模块。REF_LON=163.125与taichi-shared中POLE_SI的概念一致，同样硬编码。EARTH_RADIUS_KM常量（无论是3476还是6371）从未被任何预测模块调用。架构师已决定将地震预测作为独立仓库拆分，本文档即为独立拆分后的技术基线。

1.2 当前状态

V8.2final为生产基线，回测命中率为62%（37/60个预测点）。基线已于2026-06-13冻结，不再追加任何改动。冻结决策的依据是：经过V9.0、V9.1、V9.2、V2.0等多个候选版本的回测验证，没有任何一个后续版本在总命中率上超越V8.2final。部分版本在T1窗口有改善，但代价是T2或T3窗口的损失更大，净效果为负。

1.3 战略转向

余震预测赛道的战略价值有限。原因在于：余震预测已有Båth定律等主流经验公式作为基线，V8.2final的62%与V7.3修正大森公式的65%相比并无显著优势，20样本下的统计显著性不足（p≈0.252）。架构师孙永吉决定将项目重心转向LLM推理压缩和量子线路设计方向，主攻AFAC2026赛题四（金融长文本记忆压缩）。地震预测系统的代码和文档保留为易宇本源体系的应用验证案例，但不再投入新的开发资源。V9.3方向三（γ引擎代换t_res查表函数）已设计但搁置执行。

1.4 文件清单

系统由以下独立脚本组成，全部可独立运行，无交叉依赖：

第二部分：V8.2final完整技术规范

2.1 全局常量

eq_v82_final.py第14-15行定义了两个全局常量：

REF_LON = 163.125

V_S = (math.sin(math.radians(60))**0.6 + math.cos(math.radians(23))**0.4) * (1 - 0.034)

其中REF_LON是空间共振函数的参考经度，对应巳方阳极位置。V_S在第14行定义但从未被任何计算函数调用，属于遗留死代码，详细分析见2.11节。

eq_v82_final.py第16-20行定义了T_RES_GROUPS映射表：

BRANCH_GROUP = {0:"子午",6:"子午", 1:"丑未",7:"丑未", 2:"寅申",8:"寅申",
                3:"卯酉",9:"卯酉", 4:"辰戌",10:"辰戌", 5:"巳亥",11:"巳亥"}
GROUP_CENTER = {"子午":30,"丑未":90,"寅申":150,"卯酉":210,"辰戌":270,"巳亥":330}

这是一个12元素的字典，key为0到11的整数，value为组名和对应的中心角度。注意到index 6-11与0-5完全重复：index 6和0都是"子午"/30°，index 7和1都是"丑未"/90°，以此类推。因此T_RES_GROUPS实际上只有6个不同的映射条目，构成一个6年子周期，而非完整的12年周期。这个重复结构将在2.3节中详细讨论。

2.2 空间共振函数 r_space(lon)

eq_v82_final.py第21-23行：

def r_space(lon):
    if lon < 0: lon += 360
    return math.cos(math.radians(lon - REF_LON))

该函数接收经度值（度），首先对西经坐标做lon+360转换，然后计算经度与参考经度REF_LON之差的余弦值。数学表达式为：

$r_{space}(lon) = \cos((lon - 163.125)°)$

其中西经lon<0时先做$lon \leftarrow lon + 360$的预处理。函数值域为$[-1, +1]$。当经度恰好为163.125°时$r_{space} = 1$（最大共振），当经度为343.125°（163.125°的对径点）时$r_{space} = -1$（最小共振）。例如百慕大经度-71°经过转换后计算$\cos((-71+360) - 163.125)° = \cos(125.875°) \approx -0.586$。

2.3 时间共振函数 t_res(year)

eq_v82_final.py第24-26行：

def t_res(year):
    idx = (year - 4) % 12
    return math.cos(math.radians(GROUP_CENTER[BRANCH_GROUP[idx]] - 330))

该函数是一个查表函数，不是连续的解析表达式。计算步骤为：首先通过$idx = (year - 4) \bmod 12$得到0-11的整数索引，然后查BRANCH_GROUP获取组名，再查GROUP_CENTER获取该组的中心角度，最后计算中心角度与330°之差的余弦值。完整映射关系如下：

| index | 组名 | 中心角(°) | $t_{res} = \cos(\text{中心角} - 330°)$ |
|-------|------|-----------|---------------------------------------|
| 0 | 子午 | 30 | $\cos(-300°) = \cos(60°) = 0.5000$ |
| 1 | 丑未 | 90 | $\cos(-240°) = \cos(120°) = -0.5000$ |
| 2 | 寅申 | 150 | $\cos(-180°) = \cos(180°) = -1.0000$ |
| 3 | 卯酉 | 210 | $\cos(-120°) = \cos(120°) = -0.5000$ |
| 4 | 辰戌 | 270 | $\cos(-60°) = \cos(60°) = 0.5000$ |
| 5 | 巳亥 | 330 | $\cos(0°) = 1.0000$ |
| 6 | 子午 | 30 | $0.5000$ |
| 7 | 丑未 | 90 | $-0.5000$ |
| 8 | 寅申 | 150 | $-1.0000$ |
| 9 | 卯酉 | 210 | $-0.5000$ |
| 10 | 辰戌 | 270 | $0.5000$ |
| 11 | 巳亥 | 330 | $1.0000$ |

$t_{res}$只取四个离散值：$+1.0, +0.5, -0.5, -1.0$。其中index 0-5与6-11完全对称重复，实际存在6年子周期。以2023年为例：$idx = (2023-4) \bmod 12 = 7$，查表得"丑未"/90°，$t_{res} = -0.5000$。

2.4 构造敏感度函数 sr(lon, lat)

eq_v82_final.py第29-33行：

def sr(lon, lat):
    if 98 <= lon <= 105 and 22 <= lat <= 30: return 1.32
    if 35 <= lon <= 45 and 35 <= lat <= 42: return 1.18
    return 1.05

这是一个三档硬切函数，不是连续衰减公式。需要强调的是，早期文档（v1）曾错误地将此函数描述为$\exp(-d^2/\sigma^2)$型连续衰减，实际上源码实现为简单的if-elif-else条件分支，不存在任何距离计算或高斯衰减。三个区域对应的含义是：

98°E ≤ lon ≤ 105°E 且 22°N ≤ lat ≤ 30°N → sr = 1.32，覆盖川滇块体（含2008年汶川M7.9、2013年芦山M6.6等震例）
35°E ≤ lon ≤ 45°E 且 35°N ≤ lat ≤ 42°N → sr = 1.18，覆盖安纳托利亚板块（含2023年土耳其M7.8双震）
其余所有区域 → sr = 1.05

当输入坐标在川滇和安纳托利亚两个区域的边界上时，sr值会从1.32或1.18跳变到1.05，缺乏地理连续性。这个设计选择是有意的：在仅有20个训练样本的情况下，引入连续过渡函数会引入额外参数，增加过拟合风险。

2.5 综合场强 S

综合场强S定义为三个分量的乘积：

$S = r_{space}(lon) \times t_{res}(year) \times sr(lon, lat)$

在eq_v82_final.py第34-39行的predict函数中，S并不是一个独立的中间变量，而是通过各分量直接代入Δ公式计算的。其中$|R|$表示$r_{space}$的绝对值，$T$表示$t_{res}$的值，$S$（即代码中的小写s）表示$sr$的值。因此第36行中R的计算调用了r_space函数（第21-23行），T的计算调用了t_res函数（第24-26行），S的计算调用了sr函数（第29-33行），三者相乘后参与Δ的公式。

2.6 三窗口预测公式

eq_v82_final.py第34-39行定义了predict函数的核心逻辑：

def predict(mag, lon, lat, yr, window):
    R = r_space(lon); T = t_res(yr); S = sr(lon, lat)
    if window == "T1":
        delta = (1.0 + 0.5*abs(R)) * S - 0.15*(1-T) - 0.034
    elif window == "T2":
        delta = (1.8 + 1.2*abs(R)) * S - 0.24*(1-T) - 0.034
    else:
        delta = (1.8 + 1.0*abs(R)) * S - 0.18*(1-T) - 0.034
    return round(mag - delta, 1)

三个时间窗口的定义和完整参数如下：

T1窗口（主震后0-24小时，预测时间偏移+4小时）：
$\Delta_{T1} = (1.0 + 0.5 \times |R|) \times S - 0.15 \times (1 - T) - 0.034$

T2窗口（主震后24-72小时，预测时间偏移+32小时）：
$\Delta_{T2} = (1.8 + 1.2 \times |R|) \times S - 0.24 \times (1 - T) - 0.034$

T3窗口（主震后72-168小时，预测时间偏移+88小时）：
$\Delta_{T3} = (1.8 + 1.0 \times |R|) \times S - 0.18 \times (1 - T) - 0.034$

最终预测值：$M_{aftershock} = \text{round}(M_{main} - \Delta, 1)$

| 参数 | T1 | T2 | T3 |
|------|------|------|------|
| 时间窗口(h) | 0-24 | 24-72 | 72-168 |
| 预测偏移(h) | +4 | +32 | +88 |
| 线性系数 $a$ | 1.0 | 1.8 | 1.8 |
| $|R|$系数 $b$ | 0.5 | 1.2 | 1.0 |
| $(1-T)$系数 $c$ | -0.15 | -0.24 | -0.18 |
| 洛书修正 $d$ | -0.034 | -0.034 | -0.034 |

其中$|R| = |r_{space}(lon)|$，$T = t_{res}(year)$，$S = sr(lon, lat)$。Δ的量纲为震级单位（$M_w$），余震震级等于主震震级减去Δ。

公式结构可以理解为三部分：$(a + b|R|) \times S$是空间-构造乘积项（Δ的主体），$c \times (1-T)$是时间相位调制项（T_res越接近1时$1-T$越小，此项贡献越小），$d = -0.034$是全局洛书修正项。因此当T_res=1（巳亥年）时，时间调制项为零，Δ完全由空间构造项决定；当T_res=-1（寅申年）时，时间调制项贡献最大，T1窗口额外减去0.30，T2额外减去0.48。

2.8 最终预测公式

在eq_v82_ops.py的第157-170行，predict函数的完整实现为：

def predict(main_mag, lon, lat, year, window, tect_corr=1.0):
    R = r_space(lon)
    T = t_res(year)
    S = s_rhombus(lon, lat)
    if window == "T1":
        delta = (1.0 + 0.5 * abs(R)) * S - 0.15 * (1 - T) - 0.034
    elif window == "T2":
        delta = (1.8 + 1.2 * abs(R)) * S - 0.24 * (1 - T) - 0.034
    else:
        delta = (1.8 + 1.0 * abs(R)) * S - 0.18 * (1 - T) - 0.034
    delta *= tect_corr
    return round(main_mag - delta, 1)

注意tect_corr参数默认值为1.0（即不做修正，与纯V8.2final等价）。最终预测公式为：

$M_{aftershock} = \text{round}(M_{main} - \Delta \times f_{tectonic}, 1)$

其中$f_{tectonic}$为构造修正系数，取值见2.7节表格。

2.9 数据集

数据集包含20个全球主震案例，震级范围Mw 6.1至9.1，时间跨度2001年至2024年。构造类型覆盖环太平洋俯冲带（日本东北M9.1、智利M8.8、苏门答腊M9.1等）、大陆走滑断裂（土耳其M7.8双震、新西兰M7.0）、板内地震（中国汶川M7.9）。每个震例按3个时间窗口（T1/T2/T3）各生成一个预测点，共60个预测点。

命中标准定义为预测余震震级与实际最大余震震级之差的绝对值不超过0.5级：$|M_{pred} - M_{actual}| \le 0.5$。数据源存储于外部路径F:\地震数据\0417数据更新\data_download\下的test_eq_catalog.csv（主震目录）和test_eq_data子目录（各震例余震序列CSV），workspace内无数据副本。

2.11 V_S死代码说明

eq_v82_final.py第14行定义了V_S常量：

V_S = (math.sin(math.radians(60))**0.6 + math.cos(math.radians(23))**0.4) * (1 - 0.034)

其数值为$(\sin(60°))^{0.6} + (\cos(23°))^{0.4}) \times 0.966 \approx 1.004$。这个常量在整个eq_v82_final.py中未被任何函数引用、未被任何计算使用。它可能来自早期版本的实验性设计（尝试将黄赤交角23°和C₆旋转角60°组合成一个全局归一化因子），但在V8.2final的最终实现中被弃用，仅作为遗留代码保留。保留死代码不影响程序运行，但会误导阅读者以为V_S参与了预测计算。

第三部分：V8.2final公式物理直觉

3.1 r_space的物理含义

REF_LON = 163.125°对应干支体系中的巳方阳极位置。在易宇本源的时空编码框架中，巳方（东南偏南）是阳性能量的极值方向，与亥方（西北偏北，约343°）的阴极形成对径180°的反对称关系。经度余弦$r_{space} = \cos(lon - 163.125°)$度量的是任一地点距阳极的空间共振强度：越靠近163.125°，共振越强（余弦值越接近+1）；越靠近343.125°，共振越弱（余弦值越接近-1）。在预测公式中，$|R|$的绝对值进入Δ计算，因此只有共振强度的大小（而非方向）影响预测结果。严格来说，这个函数是一种简化的空间调制，没有考虑纬度维度和地形效应，但在全球尺度上捕捉了经度方向的基本趋势。

3.2 t_res的物理含义

t_res是干支年柱的时间相位调制。在易宇本源框架中，六十甲子的地支按三合关系分为六组（子午、丑未、寅申、卯酉、辰戌、巳亥），每组对应一个60°的中心角。$t_{res} = \cos(\text{中心角} - 330°)$将年份映射到$[-1, +1]$的时间共振强度。巳亥年（index 5, 11）$t_{res} = +1.0$（最强共振），寅申年（index 2, 8）$t_{res} = -1.0$（最弱共振）。6年子周期源于T_RES_GROUPS的index 0-5与6-11完全重复，使得12年周期退化为6年周期。这可能是有意为之（因为三合六组本身就构成6年循环），也可能是实现遗漏（本应为12个不同的映射）。

3.3 sr的物理含义

sr函数用三档硬切系数放大预测幅度。川滇块体（sr=1.32）和安纳托利亚板块（sr=1.18）是全球应力积累的两个热点区域，历史大震频率高、余震序列活跃，因此预测的余震幅度需要被放大（Δ被放大→$M_{aft}$被降低，即预测更小的余震——这意味着这些区域的实际余震比全球平均水平更大，所以预测值需要向下修正以匹配实际观测）。其余区域默认sr=1.05，略微放大，反映了全球余震活动的平均水平。严格来说，这种区域放大没有连续的空间衰减模型，而是基于经验观察的离散分档。

3.4 Δ公式的物理含义

Δ公式的本质是Båth定律的易学化扩展。经典Båth定律指出主震与最大余震的震级差约为1.0-1.2级，是一个全球统计平均值。V8.2final的Δ公式在此基础上引入三个调制维度：空间维度（通过$|R|$）、时间维度（通过$(1-T)$）、区域维度（通过sr）。因此Δ不再是固定值，而是随地点、年份、构造背景变化的动态衰减量。公式中的洛书修正项-0.034来自易宇本源的河图洛书修正体系，在数值上约为0.034（即1/7循环小数142857的简化取值），作用方向为减小Δ（即略微增大预测的余震震级）。

3.5 构造修正的物理含义

俯冲带修正系数0.85表示俯冲带余震偏弱（能量释放效率高，应力快速耗散），因此Δ被缩小（$M_{aft}$被增大，即预测更大的余震）。板内修正系数1.15表示板内余震偏强（应力集中难以释放），Δ被放大（$M_{aft}$被减小，即预测更小的余震）。走滑型1.00为基准线。正断型1.08和逆冲型0.92介于两者之间。这种分类修正抓住了不同构造背景下余震行为的定性趋势，但修正系数的具体数值（0.85、1.15等）是基于20个震例的经验标定，缺乏独立的理论推导。

4.3 V9.0五大优化逐一分析

V9.0（eq_v9_full.py）引入了五个优化方向，逐一分析其效果和否决理由：

优化一：混沌期衰减 $\alpha(t) = 1 - e^{-t/\tau}$，$\tau = 6h$。在T1窗口引入时间衰减因子，模拟震后混沌期向稳态过渡。在逆天年（$t_{res} \le -0.8$）的T1窗口中，命中率从33%提升至67%，效果显著。但T2命中率从60%崩至45%（-15个百分点），T3从75%降至70%（-5个百分点）。净效果为总体从62%降至57%。否决理由：T1的增益不足以补偿T2/T3的损失。

优化二：384爻格点调制 $R_{full} = \cos(lon-163°) \times \cos(6\varphi_{lon}) \times \cos(6\varphi_{lat})$。将V8.2的$r_{space}$替换为含C₆球面格点调制的$R_{full}$。副作用是平均$|R|$从V8.2的约0.495下降到约0.268，信号幅度被压缩近半。否决理由：高频C₆调制在空间维度引入混叠，信号被稀释而非增强。

优化三：双鱼眼拓扑距离 $Y_{eye} = 0.15 e^{-d_{阳}/5000} - 0.10 e^{-d_{阴}/5000}$。以昆仑(35°N, 80°E)为阳眼、百慕大(25°N, -70°E)为阴眼，计算haversine球面距离并做指数衰减。$Y_{eye}$的值域约为±0.12，对Δ公式的影响不足（相对Δ的1-3级量级，0.12仅占4-12%）。否决理由：幅度太小，无法产生可观测的预测差异。注意：$Y_{eye}$使用的是5000km特征距离的haversine公式，与taichi-shared中的EARTH_RADIUS_KM（3476或6371）无关。

优化四：S动态化 $S_{dynamic} = S_{年} \times S_{月} \times S_{日}$。将单一的S值拆分为年柱、月柱、日柱（朔望月相位）三个余弦分量相乘。效果是反而稀释了信号，因为月和日分量的周期性与地震活动的关联性远弱于年柱分量。否决理由：三层乘积互相抵消，净信号弱于单一年柱。

优化五：分形应力 $S_{fractal} = 1.0 + 0.32 \times e^{-n/2}$。其中$n=1$为全球板块，$n=2$为川滇/安纳托利亚次级块体。在数学上$S_{fractal}$将原sr的1.05/1.18/1.32三档替换为连续指数衰减：$n=1$时$S=1.194$，$n=2$时$S=1.094$。否决理由：参数方向正确但幅度反了——T2命中率从60%崩至45%，说明分形修正系统性地放大了Δ。

4.4 V9.1 MoE三才路由

V9.1引入混合专家（MoE）路由机制，将预测分为天/地/人三个模式，通过$r_{scale} = 0.333$对已有系数做比例缩放。三才权重为$W_T = 0.20$（天）、$W_E = 0.60$（地）、$W_H = 0.20$（人）。效果是T2从60%崩至45%，T3从75%崩至45%，总体47%（28/60）。否决理由：MoE路由引入的缩放因子破坏了V8.2final经过调优的系数平衡，三才权重的分配缺乏物理依据。

4.5 V9.2花瓣函数

V9.2尝试用连续化公式替代sr的三档硬切：$S_{petal} = 1.0 + 0.30 \times e^{-d_{petal}/2000} \times e^{-depth/80}$，其中$d_{petal}$为到最近活跃断裂的距离，depth为震源深度。连续化方向正确，但在20样本条件下，深度信息不足以稳定估计新参数（depth/80的系数）。否决理由：过拟合风险高，新参数缺乏足够的样本支撑。

4.6 V2.0 384爻γ频谱独立模型

V2.0（eq_v20.py）完全脱离V8.2的$r_{space} \times t_{res} \times sr$框架，走了一条独立的计算路径：五柱干支（年月日时+胎元）→ 先贤重排表 → 384爻相位（每爻0.9375°） → γ频谱计算 → $\Gamma_{quake}$ → 线性回归映射到余震震级。γ频谱计算涉及天网项（高斯衰减到巳亥轴最近极值）、太阳谐波项（太阳黄经的衰减修正）、月球分量（独立激发项），以及C₆空间因子$\kappa_{c6}$、地质类型修正$\kappa_{geo}$、深度权重$w_{depth}$等。

五柱干支的计算需要：年柱（以立春为界，需查立春时刻预计算表）、月柱（以节气为界，需迭代计算12个节气时刻）、日柱（儒略日计算，以2000-01-07为甲子日参考）、时柱（北京时间时辰计算）、胎元（月柱天干进一位+地支进三位）。这套计算体系的理论优美度高，但预测效果远不如V8.2final。总体命中率45%（27/60），T3窗口仅30%，大震全部miss。否决理由：理论框架复杂度高但预测能力弱，384爻映射的编码精细度与其在此数据集上的预测价值不匹配。

4.7 V7.3修正大森公式

V7.3是早期冠军版本，基于修正大森公式。核心参数为：OMORI_K_BASE = 1.8e-3，OMORI_P_BASE = 1.1，OMORI_C_BASE = 0.05，KAPPA = 0.18。修正大森公式为$dN/dt = K / (t+c)^p$，其中$K = 1.8 \times 10^{-3} \times e^{0.78M} \times (d/100)^{-0.54} \times S$，$p = 1.1 + 0.05 \times (\lambda \times 10)$，$c = 0.05 \times d^{0.3}$。

V7.3的实际冠军公式是最简三参数形式：T1的$M_{aft} = M_{main} - 1.3$，T2的$M_{aft} = M_{main} - 2.5$，T3的$M_{aft} = M_{main} - 2.4$。这本质上就是Båth定律的常数版本，无空间和时间调制。总体命中率65%（39/60），T3高达80%，但T1仅60%。V8.2final的价值在于T2=60%独立超越了V7.3的55%，说明空间-时间-构造三维度调制在T2窗口提供了额外信息。

第五部分：统计学审视

5.1 样本量限制

数据集仅包含20个主震案例，60个预测点。在零假设$H_0$: 命中概率$p = 0.5$（等同抛硬币）下，精确二项检验为$\Pr(X \ge 37 | n=60, p=0.5) \approx 0.252$。这个p值远大于0.05的显著性阈值，因此不能拒绝零假设——仅凭62%的命中率和60个样本，无法在统计学上证明V8.2final的预测能力优于随机猜测。

95%置信区间的近似计算：对于$n=60$、$p_{obs} = 37/60 = 0.617$，Wilson置信区间约为$(0.384, 0.780)$，即(38%, 82%)。这意味着即使模型的真实命中率只有38%，在60次实验中也有可能观察到62%的结果。

5.2 Båth定律基线

V7.3修正大森公式（本质上就是Båth定律的常数版本）在此数据集上的命中率为65%（39/60）。这说明Båth定律本身在这个特定数据集上就不是50%——朴素零假设$p=0.5$过于宽松。更合理的基线应该是$p_{Bath} = 65\%$，但即使如此，V8.2final的62%仍低于V7.3的65%，说明V8.2final在总体上并未超越Båth定律基线。

5.3 V8.2final的价值定位

V8.2final的真正价值在于分窗口分析。T2窗口命中率60%独立超越了V7.3的55%（+5个百分点），说明空间-时间-构造三维度调制在主震后24-72小时这个时间范围内提供了Båth定律无法捕捉的额外信息。T1窗口50%低于V7.3的60%（-10个百分点），说明V8.2final引入的复杂度在短期混沌期内反而增加了噪声。T3窗口75%低于V7.3的80%（-5个百分点），但差异不大，可能在统计波动范围内。

因此V8.2final不是V7.3的全面超越者，而是在T2窗口有所改进的互补版本。这个结论需要更大样本（至少100个震例、300个预测点）才能验证是否稳健。

5.4 置信区间总结

在$n=60$的样本量下：V8.2final 62%的95% Wilson置信区间约为(38%, 82%)，V7.3 65%的95%置信区间约为(42%, 84%)。两个置信区间大面积重叠，无法从统计上区分两个版本的差异。当前所有版本对比结论都应视为描述性统计而非推断性结论。

第七部分：已知问题与后续方向

7.1 V_S死代码

eq_v82_final.py第14行定义的V_S = (sin(60°)^0.6 + cos(23°)^0.4) × (1 - 0.034) ≈ 1.004在整个文件中未被引用。该常量可能来自早期V8.2原版的设计尝试（将黄赤交角23°和C₆旋转角60°组合为归一化因子），在V8.2final中被弃用但未删除。建议在后续维护中删除此行或添加DEPRECATED注释。

7.2 T_RES_GROUPS的6年子周期

T_RES_GROUPS的index 6-11与0-5完全重复（子午/丑未/寅申/卯酉/辰戌/巳亥各出现两次），导致t_res实际只有6年周期而非完整12年周期。这可能是有意设计（三合六组本身就构成6年循环），也可能是实现遗漏（本应为12个不同的映射条目）。从物理直觉看，地支三合确实只有6组，6年周期可能是正确的。但如果是实现遗漏，则修复方法是将index 6-11映射到不同的角度值（例如引入地支六合或六冲的6组），使12年周期完整。无论哪种解释，这个重复结构是当前t_res查表函数的固有特征，不是bug。

7.3 sr函数缺乏地理连续性

sr函数的三档硬切（1.32/1.18/1.05）在区域边界上存在跳变。例如经度105.1°、纬度22°N的sr值为1.05（不在川滇框内），而经度105.0°、纬度22°N的sr值为1.32（恰好落在川滇框内）。这种不连续性是设计选择：在20样本的条件下，引入连续过渡函数需要额外的衰减参数（如特征距离σ），增加过拟合风险。V9.2的花瓣函数尝试解决此问题但未成功。

7.4 所有常数硬编码

eq_v82_final.py和eq_v82_ops.py中的所有常数（REF_LON=163.125、洛书修正-0.034、三窗口参数等）均直接硬编码在源码中，未从taichi-shared或任何共享模块导入。这意味着如果常量定义需要修改（例如洛书修正系数更新），必须在多个文件中手动同步。在太极矩阵工具包和地震预测系统拆分为独立仓库后，这种硬编码方式实际上消除了两个项目之间的耦合风险，但也意味着失去了常量一致性的自动保障。

7.5 V9.3方向三（γ引擎代换t_res）设计但搁置

V9.3方向三的核心思路是回退到V8.2final的已验证框架（$r_{space} \times t_{res} \times sr$），仅用归一化四柱γ均值替代t_res查表函数。γ引擎来自V2.0白皮书v12的参数体系（GAMMA_BASE=1.26、SIGMA=30°），计算路径为五柱干支→先贤重排→384爻相位→γ频谱→均值。设计逻辑是t_res查表函数信息粒度粗（仅4个离散值），四柱γ均值提供更精细的时间分辨。该方案零新参数引入，是最小改动方案，风险可控。但由于战略转向（1.3节），此方案已搁置执行。

7.6 战略方向

项目重心已从地震预测转向LLM推理压缩和量子线路设计。具体方向为：AFAC2026赛题四（金融长文本记忆压缩）、LLM推理压缩算法研发、华为HiQ量子平台对接（C₆群表示→量子线路编译）。地震预测系统的代码和本文档保留为易宇本源体系的应用验证案例，展示C₆六重对称群和干支时空编码在地球物理预测任务中的实际应用。如果未来有更大规模的地震数据集（100+震例）可用，V9.3方向三值得重新评估。
